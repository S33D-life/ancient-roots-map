# Roadmap Linking

> Connect community needs to the Living Roadmap.

## Purpose

The Living Roadmap grows from real needs. This skill teaches you how to identify features that matter, link them to existing roadmap items, and write clear proposals that help prioritize development.

## When to Use

- A task asks you to link user requests to roadmap features
- You've identified a gap between current functionality and community needs
- You want to write a feature proposal

## Required Inputs

- Understanding of existing roadmap items (`/roadmap`)
- User feedback, forum discussions, or bug patterns
- Knowledge of which system area the feature affects

## Required Outputs

- Feature description with clear use case
- Link to related roadmap item (or proposal for new one)
- Impact assessment: who benefits and how
- Priority suggestion with reasoning

## Proof-of-Work Standard

Your submission must include:

1. **Feature title** — concise, descriptive name
2. **User story** — "As a [role], I want [feature] so that [benefit]"
3. **Roadmap link** — existing item slug or "new proposal"
4. **Impact analysis** — estimated reach and importance
5. **Related issues** — links to relevant bug reports or discussions

## Review Criteria

Curators will check:

- [ ] Feature addresses a real community need
- [ ] Impact analysis is grounded, not speculative
- [ ] Correctly linked to existing roadmap items where applicable
- [ ] Not a duplicate of an existing proposal
