# Bug Garden

> Find and report issues to strengthen the ecosystem.

## Purpose

Quality is a form of care. The Bug Garden is where co-creators report issues, suggest improvements, and help maintain the health of S33D's living systems.

## When to Use

- You encounter unexpected behavior in the app
- A feature doesn't match its documentation
- You want to suggest a UX improvement
- A task asks for QA testing of a specific feature

## Required Inputs

- The page/feature where the issue occurs
- Steps to reproduce the problem
- Browser and device information
- Screenshots or screen recordings when possible

## Required Outputs

A bug report containing:

- **Title** — clear, specific summary
- **Steps to reproduce** — numbered, reproducible sequence
- **Expected behavior** — what should happen
- **Actual behavior** — what actually happens
- **Severity** — cosmetic, minor, major, or critical
- **Evidence** — screenshots, console errors, network logs

## Proof-of-Work Standard

Your submission must include:

1. **Bug report ID** or link to your submitted report
2. **Reproduction confirmation** — you verified it happens consistently
3. **Environment details** — browser, viewport, auth state
4. **Suggested fix** (optional but valued) — what might resolve it

## Review Criteria

Curators will check:

- [ ] Issue is reproducible from the steps provided
- [ ] Severity assessment is accurate
- [ ] Not a duplicate of an existing report
- [ ] Report is clear enough for a developer to act on
